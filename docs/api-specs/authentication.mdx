# Authentication

Brale uses OAuth 2.0 with the `client_credentials` grant type for API authentication.

## Getting Started

To authenticate with the Brale API, you'll need:

- Client ID
- Client Secret

These credentials can be obtained from the [Dashboard](https://app.brale.xyz/settings?tab=api).

## Authentication Flow

1. Encode your credentials
2. Request an access token
3. Use the token in subsequent requests

### Example Request

```bash
curl --request POST \
  --url https://auth.brale.xyz/oauth2/token \
  --header 'Authorization: Basic ${BASE_64_OF(client_id:client_secret)}' \
  --header 'Content-Type: application/x-www-form-urlencoded' \
  --data grant_type=client_credentials
```

### Example Response

```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer",
  "expires_in": 3600
}
```

## Using the Access Token

Include the access token in the `Authorization` header of all API requests:

```bash
curl --request GET \
  --url https://api.brale.xyz/accounts \
  --header 'Authorization: Bearer ${ACCESS_TOKEN}'
```

## Token Expiration

Access tokens expire after 1 hour. You should implement token refresh logic in your application to request a new token before expiration.
